# Kaggle開発環境構築

## 構成ファイル
- **Dockerfile**: Kaggle公式イメージをベースに使用
- **docker-compose.yml**: ポートフォリオ、ボリュームマウントの定義

## 起動手順
1. `docker compose up -d`
2. `docker compose logs` でトークンを確認（初回のみ）
3. `http://localhost:8888` へアクセス

## VS Code接続設定
1. Jupyter拡張機能をインストール
2. ipynbファイルを開き、右上のコマンドパレットから「Jupyter: 既存のJupyterサーバーに接続」を選択
3. `http://localhost:8888/?token=...` を入力

## Jupyter Notebookの運用とトラブル解決
1. 最適な接続方法: VS CodeのJupyter拡張機能を利用し、既存のJupyterサーバー（http://localhost:8888）に接続するのが最善

2. 発生した不具合: VS Codeからコードが実行できず、カーネルが沈黙する状態が発生

3. 解決策: ブラウザで直接サーバーにアクセスしたことにより、無効なクッキーやセッション情報がクリアされ、通信が正常化

4. 教訓: 不具合時は「ログの確認」「ブラウザでの直接試行」「キャッシュ/クッキーのクリア」をまず行う

## 環境構築トラブルシューティングの要約
1. 問題の発生:

Dockerコンテナを利用したKaggle環境において、notebooks/titanic.ipynb上のライブラリ（pandas等）に波線が表示され、認識されない状態

2. 原因の特定:

VS CodeがホストOS（またはWSL）のPython環境を参照しており、ライブラリがインストールされているDockerコンテナ内の環境と切り離されていたことが原因

3. 実施した対策:

コンテナ情報の確認: コンテナ内のPython実行パスが /usr/local/bin/python3 であることを確認

コンテナへのアタッチ: VS Codeの「リモート ウィンドウ」機能から「実行中のコンテナーにアタッチ」を選択し、エディタ自体をコンテナ内部に接続

ディレクトリの展開: コンテナ内の作業ディレクトリである /app を開き、ファイルへアクセス

インタープリターの選択: Jupyterカーネルとして、コンテナ内の推奨環境（/usr/local/bin/python）を選択
